## 1. نظام هجرة قاعدة البيانات (migrate-system)

### المقدمة والهدف

تم تصميم نظام الهجرة (Migrations System) ليحل محل الطريقة القديمة لتحديث قاعدة البيانات والتي كانت تقوم على فحص الجداول والأعمدة في كل مرة يتم تحميل صفحة في الموقع عبر ملف `includes/connect.php`. 

**الفوائد الأساسية للنظام الجديد:**
- **زيادة سرعة النظام**: توفير وقت تنفيذ استعلامات الفحص (`SHOW COLUMNS`) المتكررة عند كل HTTP Request.
- **التنظيم والأمان**: تتبع التعديلات والنسخ بنظام الحزم (Batches)، مما يتيح لك معرفة أي تعديل تم تنفيذه وتاريخه.
- **التراجع الذكي (Rollback)**: إمكانية التراجع عن التعديلات الأخيرة تلقائياً في حال حدوث مشاكل في بيئات التطوير أو الإنتاج.

---

### هيكل المجلدات والملفات

يتكون نظام الهجرات من المكونات التالية في المشروع:

```text
focus/
├── database/
│   └── migrations/                  # مجلد يحتوي على جميع ملفات الهجرات (.sql و .php)
├── classes/
│   └── MigrationManager.php         # الكلاس المسؤول عن إدارة عمليات التحقق والتنفيذ والتراجع
├── migrate.php                      # واجهة سطر الأوامر (CLI Runner) لتشغيل الأوامر في الـ Terminal
└── migrations_ui.php                # لوحة التحكم الرسومية المتميزة لإدارة النظام عبر المتصفح
```

---

### الاستخدام عبر الويب (Web UI)

يمكنك إدارة قاعدة البيانات بشكل كامل من خلال واجهة ويب تفاعلية فائقة السرعة والجمال عبر الرابط التالي:
`http://localhost/focus/migrations_ui.php`

**أبرز مميزات واجهة الويب:**
1. **شاشة عرض الحالة (Stats Grid)**: تعرض لك فوراً إجمالي الملفات، والعمليات المعلقة، والعمليات الناجحة، ورقم آخر دفعة (Batch).
2. **الخط الزمني (Timeline)**: يعرض قائمة مرتبة زمنياً بكل الهجرات مع ألوان تعبيرية لسهولة التمييز (أخضر للمنفذ، برتقالي للمعلق).
3. **مستكشف الأكواد (Code Viewer)**: أيقونة العين تتيح لك قراءة الكود المكتوب داخل ملف الهجرة مباشرة من المتصفح قبل تنفيذه.
4. **أزرار التحكم**:
   - **تنفيذ العمليات المعلقة**: زر التفعيل لتشغيل كافة العمليات التي لم تنفذ بعد.
   - **التراجع عن آخر دفعة**: للتراجع عما تم تنفيذه في الدفعة الأخيرة (مع نافذة تأكيد حماية).
5. **مولد الملفات الذكي (Migration Generator)**: نموذج جانبي لكتابة اسم الهجرة واختيار نوعها (PHP أو SQL) ليقوم الخادم بإنشاء الملف نيابة عنك في المجلد الصحيح.
6. **شاشة سجل الخادم (Terminal Output)**: محاكي سطر أوامر يعرض لك تفاصيل الاستعلامات والأخطاء مباشرة وبشكل فوري عند الضغط على أي عملية.

### الاستخدام عبر سطر الأوامر (CLI Command)

إذا كنت تفضل استخدام الـ Command Line أو الـ Terminal المدمج في محررات الأكواد مثل VS Code، يمكنك تنفيذ العمليات عن طريق تشغيل أمر `php migrate.php` متبوعاً بأحد الخيارات التالية:

#### 1. عرض حالة الهجرات الحالية
يعرض جدولاً بأسماء الملفات، حالتها (Applied / Pending)، رقم الحزمة، وتاريخ التنفيذ:
```bash
php migrate.php status
```

#### 2. تشغيل الهجرات المعلقة
يقوم بالبحث عن كافة الملفات المعلقة وتطبيقها على قاعدة البيانات وترقية إصدار الحزمة (Batch Number):
```bash
php migrate.php up
```

#### 3. التراجع عن آخر دفعة تم تشغيلها
يقوم بالتراجع عن آخر دفعة وتطبيق دوال/استعلامات التراجع المسجلة فيها وحذف سجلاتها:
```bash
php migrate.php rollback
```

#### 4. إنشاء ملف هجرة جديد
يقوم بإنشاء ملف هجرة جديد في مجلد `database/migrations/` بالاسم المطلوب مع لاحقة التوقيت الزمني تلقائياً.
يمكنك تحديد النوع إما `php` (افتراضي) أو `sql`:
```bash
# لإنشاء ملف هجرة بصيغة PHP
php migrate.php make describe_your_changes php

# لإنشاء ملف هجرة بصيغة SQL
php migrate.php make describe_your_changes sql
```

### كتابة ملفات هجرة جديدة (Writing Migrations)

يدعم النظام نوعين من ملفات الهجرة لتلائم كافة السيناريوهات:

#### أولاً: ملفات PHP (موصى بها)
تسمح لك بكتابة منطق معقد بالـ PHP مثل التحقق من الجداول أو إدراج بيانات تلقائية (Seeding) بالإضافة لتحديد دالة التراجع بدقة.

*مثال لملف هجرة PHP:*
```php
<?php

return new class {
    /**
     * تشغيل الهجرة (تعديل الجداول / إضافة أعمدة)
     */
    public function up($conn) {
        $conn->query("ALTER TABLE employees ADD COLUMN status VARCHAR(20) DEFAULT 'active'");
    }

    /**
     * التراجع عن الهجرة (حذف الأعمدة / إرجاع الجداول لحالتها السابقة)
     */
    public function down($conn) {
        $conn->query("ALTER TABLE employees DROP COLUMN status");
    }
};
```

#### ثانياً: ملفات SQL
أبسط وأسرع لكتابة استعلامات SQL نقية مباشرة. عند استخدام هذا النوع، يقوم المولد بإنشاء ملفين:
1. ملف رئيسي بصيغة `.sql` لتنفيذ التحديثات (مثال: `2026_06_09_123000_create_logs_table.sql`).
2. ملف مرافق بصيغة `.down.sql` لكتابة أوامر التراجع (مثال: `2026_06_09_123000_create_logs_table.down.sql`).

*ملاحظة: تأكد من فصل الاستعلامات المتعددة داخل ملفات الـ SQL باستخدام الفاصلة المنقوطة (`;`).*
